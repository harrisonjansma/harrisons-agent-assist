# Promo Code Problems

Use this when a discount or promo code won't apply, or a customer expected a discount that isn't showing.

## Check the code
1. Ask for the exact code and where they entered it. Codes are case-insensitive but whitespace and look-alike characters (O vs 0, l vs 1) cause failures.
2. Look up the code in **Marketing → Promotions** and confirm it is active and not expired.
3. Check the code's rules:
   - **Minimum spend** not met.
   - **New customers only** — the account has prior purchases.
   - **Single-use** or **once per customer** — already redeemed.
   - **Plan/product restrictions** — the code only applies to specific plans or items.
   - **Region restrictions** — not valid in the customer's country.
   - **Stacking** — most codes can't combine with another active discount.

## Fixing it
- If the code is valid and the customer qualifies but it still fails, apply the discount manually from **Billing → Adjustments** and note the code.
- If the customer just missed a now-expired promo by a short window, you may honor it as a one-time courtesy within reason; log the reason.
- If they don't qualify (e.g., existing customer on a new-customer code), explain plainly and, where available, offer any promotion they *are* eligible for.

## What to tell the customer
- State clearly whether the code applied and the final price.
- If manually adjusted, confirm the new amount and next billing date.

## Escalate when
- A code is broken for many customers (likely a configuration bug) — flag to Marketing.
- The customer is disputing a charge because a promised discount never applied.
