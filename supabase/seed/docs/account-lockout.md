# Account Login and Lockout

Use this procedure when a customer can't log into online or mobile banking, is locked out, or didn't get a verification code.

## Why lockouts happen
- Too many failed password attempts temporarily locks the login for security.
- A verification (one-time) code sent to a stale phone or email can't be received, blocking sign-in.

## Steps
1. Verify identity, then confirm the username and the contact on file for verification codes.
2. For a forgotten password, trigger a reset to the verified email or phone; the link or code is time-limited.
3. If the account is locked from failed attempts, release the lock after verification and have them reset the password.

## Code didn't arrive
- Confirm the phone number or email is current; update it if the customer no longer controls it (verify carefully first).
- Check spam for emailed codes and confirm the number can receive SMS.

## Escalate when
- The customer can't pass verification because all contact methods on file are outdated.
- Repeated lockouts suggest someone else is attempting to access the account.
