# Storefront SEO and Metadata

Use this procedure when a creator wants their storefront to rank or preview better — page titles, descriptions, or social share cards look wrong.

## What we control
- Each storefront has an SEO title, meta description, and an Open Graph image used for link previews.
- We auto-generate a sitemap and serve it to search engines behind Cloudflare.

## Steps
1. In **Storefront → SEO**, have the creator set the page title (under ~60 characters) and meta description (under ~155).
2. Upload or confirm the Open Graph image — this is what shows when the link is shared on social.
3. Save; changes are live immediately, but search engines re-crawl on their own schedule.

## Common issues
- **Share preview shows old image:** the social platform cached it — use that platform's URL debugger to refresh.
- **Not appearing in search:** new storefronts take days to index; confirm the storefront isn't set to private.

## Escalate when
- The sitemap returns an error or search engines report crawl failures on the domain.
