# Travel Notifications

Use this procedure when a customer is traveling and wants to avoid having their card declined for unusual location activity.

## Why it helps
- Risk monitoring may flag out-of-area or foreign transactions as suspicious and decline them.
- A travel notice tells the system to expect activity in those locations during set dates.

## Steps
1. In **Card → Travel notice**, enter the destinations (countries or regions) and the start and end dates.
2. Confirm the card's contact number is current so any verification alert reaches the customer.
3. Remind them foreign-transaction fees, if any, still apply per the account terms.

## While traveling
- Advise carrying a backup payment method in case a transaction still triggers a verification step.
- If a charge is declined, responding to the confirmation alert usually clears it immediately.

## Escalate when
- The customer is already abroad with a card that keeps declining despite an active travel notice.
