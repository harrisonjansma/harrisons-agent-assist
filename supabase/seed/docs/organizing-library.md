# Organizing Products with Folders and Tags

Use this procedure when a creator asks how to organize a large library or manually tag products into posts.

## Structure
- **Folders** group captured items (by campaign, brand, season — the creator's choice). An item can live in one folder.
- **Tags** are cross-cutting labels; an item can carry many. Use tags for themes that span folders ("gift guide", "under $50").
- **Manual product tagging** attaches specific products to a post or storefront section so it becomes shoppable.

## Common tasks
1. **Bulk organize:** multi-select in the library and apply a folder or tags in one action.
2. **Build a shoppable post:** create the post, then tag products into it; tagged products render as shoppable links through `/r/<code>`.
3. **Find things fast:** filter the library by folder, tag, or merchant.

## Notes
- Organization is private; it doesn't change what the public storefront shows until items are added to it.
- Enrichment (image, price, title) comes from capture — see *Capturing Products with the Extension*.

## Escalate when
- Filtering returns clearly wrong results after a large import.
